# Positioning and Business

## 10. Competitive Landscape and Positioning

The closest alternatives cover memory, session search, agent reflection, skill
generation, or observability. Autophagy’s defensible category is the full
verified-improvement loop.

| Competitor category | Examples | Overlap | Autophagy distinction |
| --- | --- | --- | --- |
| Learning memory | Hindsight | Retain, recall, reflect, evolve beliefs. | Compile insight into tested, versioned behavioral changes. |
| Coding-history search | ctx | Index local coding-agent sessions into SQLite for retrieval. | Detect repetition and permanently improve future behavior. |
| Persistent memory | Claude-Mem, AgentMemory | Capture, compress, and reinject session context. | Replay, shadow validation, false-positive measurement, and mutation lifecycle. |
| Self-evolving skills research | EvoSkill and related systems | Generate and refine reusable skills from failures. | Messy real-world local developer history, multi-agent product UX, and continuous background operation. |
| Native agent platforms | Claude Code and future equivalents | Hooks, skills, memory, session ownership, distribution. | Cross-agent, model-independent, auditable, local shared learning layer. |
| Agent observability | LangSmith, Phoenix-type platforms | Traces, evaluators, feedback, insights. | Improve employees’ local coding-agent workflows rather than a hosted agent product. |
| Desktop work memory | Pieces-like products | Local workflow capture and recall. | Understand agent behavior and change it, rather than remembering broad work context. |

> **Positioning line**
>
> Autophagy turns repeated agent behavior into tested, versioned, and reversible
> improvements.

## 11. Open-Source and Monetization Strategy

### 11.1 Open-source core

The complete single-user local product should be genuinely open source, ideally
under Apache-2.0: Rust engine, CLI, macOS app, adapters, event protocol, mutation
format, MCP server, and replay framework. Trust is impossible if a closed daemon
silently reads every transcript, command, and repository.

### 11.2 Paid individual tier

| Tier | Indicative price | Value |
| --- | --- | --- |
| Autophagy Core | $0 | Local engine, CLI, Mac app, unlimited local history, search, pattern detection, manual mutations, MCP, BYOK models. |
| Autophagy Plus | $8–12/month or ~$79/year | Signed auto-updating builds, encrypted sync, automatic model management, advanced replay, scheduled digestion, backups, priority adapters, analytics. |
| Autophagy Teams | $20–30/user/month | Shared team genome, private registry, cross-developer patterns, runbooks, approvals, policies, redaction, impact analytics. |
| Enterprise | Annual contract | Self-hosted and air-gapped deployment, SSO, audit logs, retention, custom adapters, private endpoints, support. |
| Marketplace | Revenue share later | Paid, tested mutation packs after trust and package standards mature. |

### 11.3 Sustainable moat

- The quality of replay and evaluation.
- A polished official macOS experience.
- Breadth and reliability of adapters.
- The mutation package ecosystem and compatibility standard.
- Team learning, governance, and encrypted synchronization.
- Accumulated product taste around useful versus noisy interventions.
